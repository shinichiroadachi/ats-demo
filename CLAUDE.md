# ATSデモ プロジェクト

中小企業向け採用管理システム(ATS)のデモ版を開発するプロジェクト。
経営者にデモを見せてフィードバックを引き出すことが目的。本番運用は想定しない。

## 最初に読むこと

- `docs/ATS_DEMO_SPEC.md` — 要件定義書。画面構成・データモデル・スコープの正。
- `supabase/schema.sql` — DBスキーマ(要件定義と同内容)
- `supabase/seed.sql` — デモ用ダミーデータ

## 開発ルール

- スタック: Next.js (App Router) + TypeScript + Tailwind CSS + Supabase
- UIはすべて日本語
- 見た目の完成度と操作の気持ちよさを最優先(デモ用途のため)
- スコープ外の機能(メール通知・日程調整・認証・媒体連携)は実装しない
- Supabase接続情報は `.env.local` に置く(`.env.example` を参照)

## 実装順序

1. Next.jsプロジェクトのセットアップ(このリポジトリ直下に作成)
2. Supabaseスキーマ適用+シード投入
3. カンバンボード(候補者パイプライン)← 最重要。D&Dの体験に一番時間をかける
4. 候補者詳細(モーダル/サイドパネル)
5. 求人管理
6. ダッシュボード
7. デザイン全体調整

## 進め方

- 各ステップ完了ごとに動作確認を挟むこと
- 大きな設計判断(ライブラリ選定の変更など)は要件定義から逸れる場合のみ確認を取ること
